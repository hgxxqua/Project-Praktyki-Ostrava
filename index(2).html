Детальный план

Нажми на шаг — раскроется что делать и кто отвечает.

01
База данных — 4 таблицы Егор
users · products · orders · order_items
▼
users — id, name, email, password (хеш), role (user/admin), created_at
products — id, name, description, price, size, brand, category, stock
orders — id, user_id, total, created_at
order_items — id, order_id, product_id, quantity, price
Категории в products: sport, everyday, formal, kids, sale
CREATE TABLE Типы данных MySQL
02
Подключение — db.php Егор
Одна функция getDB() — все берут отсюда
▼
Один файл db.php — только здесь хранятся логин, пароль и имя БД
Функция getDB() через PDO — безопасная работа с запросами
Все остальные файлы делают только require_once 'db.php'
PDO PDO::prepare
03
Функции — functions.php Егор
Весь SQL только здесь — остальные только вызывают
▼
Товары: getAllProducts(), getProductById(), getProductsByCategory(), addProduct(), deleteProduct()
Пользователи: registerUser(), loginUser()
Корзина: addToCart(), getCart(), removeFromCart(), clearCart(), getCartTotal()
Заказы: createOrder(), getOrdersByUser()
Помощники: isLoggedIn(), isAdmin(), redirect()
Функции PHP Массивы
04
Аккаунты — $_POST + $_SESSION Ренат
register.php · login.php · logout.php
▼
register.php — GET показывает форму, POST вызывает registerUser()
login.php — вызывает loginUser(), пишет user_id, name, role в $_SESSION
logout.php — session_destroy(), редирект на главную
Пароль только через password_hash() — никогда не хранить открытым
После входа role = "user" или "admin" — это основа всей системы прав
$_SESSION password_hash $_POST / $_GET
05
Витрина — $_GET + категории Эрик
index.php · product.php · фильтр по ?category=
▼
index.php — вызов getAllProducts() или getProductsByCategory() если есть ?category=
product.php — берём ?id=5 из $_GET, вызов getProductById()
Меню категорий — ссылки с параметром ?category=sport и т.д.
Кнопка "В корзину" — форма с POST на cart.php
Только текст — без фото, максимально просто
$_GET
06
Корзина — $_SESSION Эрик
cart.php — добавить, удалить, оформить
▼
Корзина = массив в $_SESSION: ключ = id товара, значение = количество
Добавить — POST → addToCart(). Удалить — POST → removeFromCart()
Итого — getCartTotal() считает сумму по всем позициям
Оформить — createOrder(), потом clearCart()
Корзина работает и без входа в аккаунт
$_SESSION
07
Админ-панель — только для admin Денис
admin.php · cabinet.php · проверка isAdmin()
▼
Первая строка admin.php — if(!isAdmin()) redirect('index.php')
Форма: название, описание, цена, размер, бренд, категория, остаток
POST → addProduct() → редирект обратно
Список товаров с кнопкой удалить → deleteProduct()
В навбаре ссылка на Admin — показывается только если isAdmin()
cabinet.php — для user: история заказов через getOrdersByUser()
include/require

Роли

User vs Admin

Что видит каждый после входа в аккаунт.

role = "user"

Обычный покупатель

  • Просмотр каталога по категориям
  • Карточка товара с описанием и ценой
  • Добавление в корзину
  • Оформление заказа
  • Личный кабинет с историей заказов
  • Нет ссылки на Admin Panel в меню
  • Нет кнопок добавления/удаления товаров
  • Прямой URL /admin.php → редирект на главную
role = "admin"

Администратор

  • Всё что есть у обычного пользователя
  • Ссылка "⚡ Админ" в навигации
  • Добавление товаров через форму
  • Удаление товаров из каталога
  • Управление остатками на складе
  • Просмотр всех заказов
  • Изменение любого товара
