Правило №1 — читать всей командой

База данных — общая карта проекта. Все таблицы и поля называются одинаково всеми участниками. Всё что повторяется — оформляется как функция в functions.php. Никто не пишет SQL прямо в страницах.

Шаги

План разработки

01
База данных — создаём таблицы
users · products · orders · order_items
▼
users — id, имя, email, пароль (хеш!), роль (user / admin), дата регистрации
products — id, название, описание, цена, размер, бренд, категория, остаток на складе
orders — id, кто сделал заказ (user_id), итоговая сумма, дата
order_items — id, к какому заказу (order_id), какой товар (product_id), количество, цена
Все имена таблиц и полей согласовываются командой до начала кодинга
MySQL — SQL команды CREATE TABLE
02
Подключение к БД — файл db.php
Один файл, одна функция getDB()
▼
Один файл db.php — всё подключение только здесь
Одна функция getDB() — все остальные файлы вызывают только её
Используем PDO — безопасно, защита от SQL-инъекций
Настройки хоста, имени БД, логина и пароля только здесь
php.net — PDO PDO::prepare()
03
Все функции — файл functions.php
Товары · Пользователи · Корзина · Помощники
▼
Товары: getAllProducts(), getProductById(), getProductsByCategory(), addProduct(), deleteProduct()
Пользователи: registerUser(), loginUser()
Корзина: addToCart(), getCart(), removeFromCart(), clearCart(), getCartTotal()
Помощники: isLoggedIn(), isAdmin(), redirect()
Никакого SQL в страницах — только вызовы функций
php.net — Функции php.net — Массивы
04
Регистрация и вход — $_POST + $_SESSION
register.php · login.php · logout.php
▼
register.php — GET показывает форму, POST сохраняет пользователя в БД
login.php — проверяет пароль, записывает user_id, name и role в $_SESSION
logout.php — уничтожает сессию, редирект на главную
Пароль НИКОГДА не хранится открытым — только через password_hash()
После входа: role = "user" или role = "admin" — это ключ ко всей системе прав
php.net — Сессии password_hash() $_POST / $_GET
05
Витрина с категориями — $_GET
index.php · product.php · категории в URL
▼
index.php — список товаров, фильтр по категории через ?category=sport
product.php — карточка товара, берём id через ?id=5 из $_GET
Категории в таблице products: sport, everyday, formal, kids, sale
Меню с категориями — ссылки с параметром ?category=…
Только текст, без фото — всё максимально просто
$_GET в PHP
06
Корзина — хранится в $_SESSION
cart.php — добавить, удалить, оформить
▼
Корзина — массив в $_SESSION: ключ = id товара, значение = количество
Добавить товар — POST запрос → addToCart()
Показать корзину — getCart() + getCartTotal()
Удалить позицию — removeFromCart(), очистить всё — clearCart()
Оформить заказ — записать в orders и order_items, затем clearCart()
Корзина не требует входа — работает и для гостей
$_SESSION
07
Админ-панель — только для admin
admin.php — добавление и удаление товаров
▼
В самом начале admin.php — проверка isAdmin(), иначе редирект на главную
Форма добавления товара: название, описание, цена, размер, бренд, категория, остаток
POST → addProduct() → редирект обратно в админку
Список всех товаров с кнопкой удалить
Обычный пользователь (role = "user") не видит кнопки и ссылки на админку в GUI

Как работает

Путь пользователя после входа

Пользователь заполняет форму входа
login.php получает email и пароль через $_POST
Проверка через loginUser()
Функция ищет пользователя в БД, проверяет хеш пароля через password_verify()
Данные записываются в $_SESSION
user_id, name, role → сессия запомнит это на всё время визита
Каждая страница проверяет роль
isLoggedIn() и isAdmin() читают из $_SESSION — никаких запросов к БД
GUI отличается по роли
user — обычное меню. admin — дополнительная ссылка на панель управления
Выход — logout.php
session_destroy() — сессия удалена, пользователь снова гость
